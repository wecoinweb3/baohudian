# 参考图智能引导方案

## 目标

当用户上传参考图但没有明确说明修改要求时，系统不直接出图，而是先进入“智能澄清意图”阶段。

## 适用场景

- 已上传参考图，输入为空后点击发送
- 已上传参考图，但输入仅为“参考这个”“帮我改一下”“按这个来”等模糊表达

## 设计原则

1. **规则负责判断是否需要追问**
   - 是否上传了参考图
   - 是否缺少足够明确的修改要求
2. **大模型负责生成更自然的引导问题**
   - 生成一句追问
   - 生成 2~4 个建议选项
   - 生成一句示例输入
3. **模型只引导，不替用户决定流程**
   - 用户仍需点击建议项或手动补充描述

## 状态分流

- `reference_unclear`：参考图已上传，但需求不明确
- `reference_edit`：保留版式，修改局部文字/参数
- `reference_restyle`：只参考风格，重新生成

## 建议接口

`POST /api/generate/clarify-reference-intent`

### 请求

```json
{
  "prompt": "参考这个图",
  "messages": [{ "role": "user", "content": "..." }],
  "referenceImages": [{ "id": "1", "name": "参考图.png", "src": "data:image/..." }]
}
```

### 返回

```json
{
  "success": true,
  "question": "我识别到你上传了一张参考图。你是想保留这个版式，只修改其中的文字内容，还是参考这个风格重新生成一版？",
  "suggestions": [
    "保留版式，只改文字",
    "参考这个风格重新生成",
    "提取图中文字后让我修改",
    "只参考配色和排版"
  ],
  "exampleInput": "例如：把公司名称改成“爱家空间设计”，电话改成“15715987903”，其他尽量不变。"
}
```

## 第一版前端交互

1. 在首页新增“添加参考图”按钮
2. 参考图与 Logo 分开管理
3. 若参考图已上传且文本不明确，发送时调用智能引导接口
4. 将返回结果展示为：
   - 一条引导消息
   - 若干建议按钮
   - 一句示例输入
5. 用户点击建议按钮后，将建议文本带入输入框，继续补充需求

## 第一版后端实现

1. 只做“意图澄清”，不直接做参考图改版
2. 大模型输入包含：
   - 用户输入
   - 最近若干轮聊天上下文
   - 参考图数量/文件名
3. 输出严格限制为 JSON

## 后续扩展

1. 参考图 OCR / 版式分析
2. 参考图改字（保留版式）
3. 参考图仿版重制（参考风格重新生成）
