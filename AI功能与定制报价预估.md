# 保护垫智能设计项目 AI 功能与定制报价预估

> 适用前提：本项目为**单客户定制化开发项目**，不是 SaaS 平台；文本大模型优先采用 **DeepSeek** 或 **通义千问 Qwen**，不采用 GPT / OpenAI 作为默认报价模型。

---

## 1. 当前首页涉及 AI 大模型的功能点

当前首页入口为 `/`，主要页面为 `src/pages/ChatSetupPage.tsx`，核心形态是“保护垫智能设计助手”的对话式生成流程。

### 1.1 文本需求生成保护垫设计方案

用户输入尺寸、颜色、标题、热线、提示语、logo 使用方式等需求后，系统调用文本大模型生成结构化画布 JSON。

当前项目不是直接让 AI 生成图片，而是：

```text
用户需求 → 大模型生成结构化 JSON → 前端本地渲染 PNG 预览图
```

对应功能：

- 理解用户自然语言需求；
- 生成保护垫尺寸、安全区域、背景色；
- 生成标题、副标题、热线、安全提示、色块、图片等元素布局；
- 输出可渲染的 JSON 草稿；
- 前端再合成图片供用户下载。

### 1.2 参考图理解与用户意图判断

当用户上传参考图，但输入比较模糊，例如：

- “参考这个”；
- “按这个来”；
- “帮我改一下”；
- “照这个做”。

系统会调用视觉模型识别参考图，并判断用户更可能想要：

- 保留版式，只改文字；
- 参考风格重新生成；
- 提取图中文字后再修改；
- 只参考配色和布局。

该功能需要使用支持图片理解的模型，例如 **qwen-vl-plus**。

### 1.3 参考图文字提取与提示词整理

当用户上传参考图并输入：

- “提取图中文字，让我修改”；
- “根据图片文字重新制作”；
- “提取图片中的文字后重新排版”。

系统会调用视觉模型：

- 识别参考图中的标题、副标题、电话、安全提示、色标说明等；
- 提取关键文字；
- 整理成一段可编辑的设计提示词；
- 用户确认或修改后，再正式生成保护垫设计图。

### 1.4 复杂排版微调

用户生成一版后，可以继续说：

- “logo 往右一点”；
- “标题更大”；
- “背景改蓝色”；
- “整体更接近参考图”；
- “logo 和标题层级再协调一点”。

当前项目有两种处理方式：

#### 本地规则微调

简单、明确的微调由本地规则处理，不消耗大模型 token，例如：

- logo 左移 / 右移 / 上移 / 下移；
- logo 放大 / 缩小；
- 标题变大 / 变小 / 加粗；
- 背景改红 / 蓝 / 黄；
- 热线条加长 / 缩短。

#### AI 复杂微调

本地规则无法识别的复杂指令，会调用文本模型生成 JSON patch，再由前端应用到当前画布草稿中。

### 1.5 Logo 预处理

当前项目已有基础 logo 处理接口，主要能力包括：

- 去背景；
- 单色化；
- 扁平化；
- 转成适合印刷的颜色。

目前该能力主要通过本地 Python 脚本实现，通常不消耗大模型 token。

如果客户要求更高质量 logo 抠图，例如复杂背景、透明 PNG、边缘优化、批量抠图，则建议单独作为增强功能报价。

## 2. 推荐模型选择

### 2.1 DeepSeek

DeepSeek 适合承担文本类任务：

- 用户需求理解；
- 生成结构化 JSON；
- 排版规则推理；
- 复杂微调指令理解；
- 生成简短回复。

推荐模型：

| 模型 | 适合程度 | 说明 |
|---|---:|---|
| deepseek-chat / DeepSeek-V3 | 高 | 性价比高，适合文本生成 JSON |
| deepseek-reasoner / R1 | 中 | 推理强，但延迟和成本更高，没必要每次使用 |

注意：DeepSeek 文本模型不适合直接做图片理解。如果需要参考图识别、OCR、图片内容理解，需要额外搭配视觉模型或 OCR 服务。

### 2.2 通义千问 Qwen

通义千问适合同时处理文本和视觉任务。

推荐模型：

| 模型 | 适合功能 |
|---|---|
| qwen-turbo | 普通文本、低成本生成 |
| qwen-plus | 文本生成 JSON、对话理解、排版描述 |
| qwen-vl-plus | 参考图理解、图片 OCR、视觉分析 |
| qwen-vl-max | 更强视觉理解，成本更高 |

如果项目中参考图、OCR、图片理解较多，建议使用：

```text
文本生成：DeepSeek 或 qwen-plus
图片理解：qwen-vl-plus
```

## 3. 反复微调 20 次生成满意图的成本估算

### 3.1 当前项目的实际生成方式

当前项目的主要流程不是直接 AI 生成图片，而是：

```text
文本/视觉模型负责理解需求和生成排版 JSON
前端负责本地渲染成 PNG 图片
```

因此模型成本主要来自：

- 文本输入 token；
- 输出 JSON token；
- 如有参考图，则增加图片理解成本；
- 如有高质量 logo 抠图，则增加抠图 API 或图像模型成本。

---

## 4. 纯文本生成，不带参考图

适用场景：用户只输入文字需求，例如：

> 帮我做一个 120×70 的保护垫，白底红字，标题是 XXX，下面放电话和安全提示。

### 4.1 单次完整生成 token 估算

| 项目 | 估算 |
|---|---:|
| 系统提示词 | 1,500 - 3,000 token |
| 用户需求 + 历史上下文 | 500 - 2,000 token |
| 输出 JSON | 1,000 - 2,500 token |
| 单次合计 | 3,000 - 7,500 token |

### 4.2 20 次完整重新生成 token 估算

| 项目 | 估算 |
|---|---:|
| 输入 token | 40,000 - 100,000 |
| 输出 token | 20,000 - 50,000 |
| 总 token | 60,000 - 150,000 |

## 5. DeepSeek 成本估算

以下为常见 API 价格量级粗估，实际价格以服务商最新报价为准。给客户报价时建议预留 3 - 5 倍余量。

### 5.1 DeepSeek Chat / V3 价格量级

| 类型 | 价格量级 |
|---|---:|
| 输入 token | 约 ¥1 - ¥2 / 100 万 token |
| 输出 token | 约 ¥4 - ¥8 / 100 万 token |

### 5.2 20 次完整生成成本估算

| 项目 | 估算 |
|---|---:|
| 输入 40,000 - 100,000 token | 约 ¥0.04 - ¥0.20 |
| 输出 20,000 - 50,000 token | 约 ¥0.08 - ¥0.40 |
| 合计 | 约 ¥0.12 - ¥0.60 |

如果只是首次生成 + 后续 19 次本地快捷微调：

```text
约 ¥0.01 - ¥0.05
```

如果 20 次里有一半走 AI 复杂微调：

```text
约 ¥0.10 - ¥0.50
```

结论：使用 DeepSeek 做文本排版生成时，单个客户生成一张满意图的模型成本通常可以按 **1 元以内** 估算。

## 7. 带参考图 / 图片 OCR 的成本估算

如果客户会经常上传参考图，成本主要来自视觉模型。

### 7.1 一次参考图流程通常包括

1. 上传参考图；
2. 视觉模型识别用户意图；
3. 视觉模型提取图中文字；
4. 用户修改提示词；
5. 文本模型生成设计 JSON。

### 7.2 单次参考图流程成本估算

| 功能 | 推荐模型 | 成本估算 |
|---|---|---:|
| 判断参考图意图 | qwen-vl-plus | ¥0.01 - ¥0.10 |
| 提取图中文字 | qwen-vl-plus / OCR | ¥0.02 - ¥0.20 |
| 生成设计 JSON | DeepSeek / qwen-plus | ¥0.01 - ¥0.10 |
| 单次参考图完整流程 | 混合 | 约 ¥0.05 - ¥0.50 |

### 7.3 20 次都带参考图的估算

| 情况 | 估算成本 |
|---|---:|
| 每次都重新分析参考图 | ¥1 - ¥10 |
| 只首次分析参考图，后续复用结果 | ¥0.20 - ¥2 |

建议实现时对参考图识别结果做缓存，不要每次微调都重新调用视觉模型。这样可以降低成本并提高响应速度。

## 8. Logo 抠图功能成本与报价建议

Logo 抠图是否使用大模型，取决于客户要求。

### 8.1 方案一：本地算法处理

适合：

- 简单 logo；
- 白底图；
- 对边缘要求不高；
- 只需要转成红色、黑色、蓝色等印刷色。

成本：

```text
API 成本接近 0
```

开发报价建议：

```text
¥1,500 - ¥3,000
```

### 8.2 方案二：接第三方抠图 API

适合：

- 客户上传各种复杂 logo；
- 背景不固定；
- 要透明 PNG；
- 要边缘更干净。

成本估算：

| 项目 | 估算 |
|---|---:|
| 单张抠图 API 成本 | ¥0.02 - ¥0.30 / 张 |
| 一个客户处理 20 张 logo | ¥0.40 - ¥6 |

开发报价建议：

```text
¥3,000 - ¥6,000
```

### 8.3 方案三：本地部署抠图 / 分割模型

可选模型方向：

- RMBG；
- MODNet；
- U2Net；
- 其他开源背景分割模型。

优点：

- 后续单张成本低；
- 不依赖第三方接口；
- 适合私有化部署。

缺点：

- 部署复杂；
- 服务器配置要求更高；
- 效果需要调参和测试。

开发报价建议：

```text
¥6,000 - ¥15,000
```

如果进一步包含：

- 自动裁边；
- 自动居中；
- 自动单色化；
- 自动转印刷红 / 蓝 / 黑；
- 自动适配保护垫安全区域；

建议报价：

```text
¥10,000 - ¥20,000
```

## 9. 单客户定制项目报价方式

由于本项目不是 SaaS，不建议按大量用户并发、月活或长期 token 池来报价。

## 11. 给客户的说明话术

可以这样向客户解释：

> 这个系统不是简单调用 AI 画图，而是为保护垫设计场景定制了一套“对话理解 → 版式生成 → logo 处理 → 本地高清渲染 → 人工微调”的流程。AI 主要负责理解需求、生成排版结构、识别参考图和辅助微调，最终图片由系统本地渲染，成本更低、结果更稳定，也更适合印刷类设计。

关于 AI 成本，可以这样说明：

> 如果只使用 DeepSeek / 千问文本模型，单次生成和反复微调 20 次的接口成本通常在 1 元以内；如果加入参考图识别、OCR、logo 高质量抠图，单张满意图的综合接口成本一般在 1 - 10 元。实际项目报价主要取决于定制功能复杂度、交付质量、部署和维护，而不是 token 成本。

在 DeepSeek / 千问方案下，单个用户反复微调 20 次，生成一张满意保护垫设计图，大致成本如下：

| 使用方式 | 单张满意图成本 |
|---|---:|
| 纯文本生成 + 本地微调 | ¥0.10 - ¥1 |
| 20 次都走文本 AI 微调 | ¥0.50 - ¥2 |
| 带参考图识别 + OCR | ¥1 - ¥5 |
| 带参考图反复识别 20 次 | ¥3 - ¥10 |
| 加第三方 logo 抠图 | 每张额外 ¥0.02 - ¥0.30 |
| 高质量图像模型 / 抠图服务 | 每张额外 ¥0.50 - ¥3+ |

## 13. 最终建议

对这个单客户定制项目来说：

1. AI 接口成本不是主要成本；
2. 主要报价应体现在定制开发、业务流程打磨、logo/参考图处理、部署和售后维护上；
3. 文本模型建议使用 DeepSeek 或 qwen-plus；
4. 参考图识别建议使用 qwen-vl-plus；
5. Logo 抠图如果只是基础处理，可以本地算法解决；如果要求高质量透明 PNG，应单独报价。

推荐最终报价口径：

| 版本 | 建议报价 |
|---|---:|
| 最小可交付版 | ¥12,000 - ¥18,000 |
| 实用生产版 | ¥25,000 - ¥40,000 |
| 高质量增强版 | ¥45,000 - ¥80,000 |
