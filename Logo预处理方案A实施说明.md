# Logo预处理方案A实施说明

## 目标

针对用户上传的 Logo 图片，提供一条**不依赖图像大模型**的基础预处理链路，让系统自动输出更适合保护垫印刷预览的扁平化 Logo。

本方案优先解决以下问题：

- 原图存在背景纹理、拍摄底色、阴影、高光
- Logo 为金属字、浮雕字、立体高光，不适合直接上版
- 当前生成结果中图片元素虽然能放到画布里，但视觉仍偏“实拍图”，不够像印刷稿

## 方案选择

采用 **方案 A：规则化预处理**。

不直接调用 `gpt-image-2`，原因是：

1. Logo 更强调**轮廓一致性**与**品牌识别稳定性**
2. 直接生成/重绘容易让图形比例、转角、细节跑偏
3. 当前目标是“印刷预览可用”，规则化处理更稳定、成本更低、可控性更强

## 实现原则

### 1. 先清洗，再上版

用户上传的 Logo 不直接进入画布，而是先经过预处理：

1. 去背景
2. 降低阴影/高光干扰
3. 单色化
4. 输出透明 PNG
5. 再作为画布中的 `image` 元素使用

### 2. 输出以“印刷可视化”为目标

当前阶段不追求精细矢量化，而是先保证：

- 背景透明
- 颜色统一
- 视觉扁平
- 能稳定用于首页效果图展示

## 后端处理流程

新增接口：

- `POST /api/generate/logo-normalize`

请求参数：

```json
{
  "image": "data:image/png;base64,...",
  "fileName": "logo.png",
  "targetColor": "#ef0000"
}
```

返回参数：

```json
{
  "success": true,
  "imageUrl": "data:image/png;base64,...",
  "fileName": "logo.png",
  "info": {
    "width": 512,
    "height": 512,
    "targetColor": "#ef0000",
    "steps": ["去背景", "单色化", "扁平化"]
  }
}
```

### 处理步骤

#### 第一步：背景识别

使用四角像素均值估算背景主色，适合当前常见场景：

- 白底
- 浅灰底
- 纹理较轻的浅色背景

#### 第二步：前景提取

按像素与背景色距离生成透明度掩码：

- 与背景接近 → 透明
- 与背景差异较大 → 保留

#### 第三步：单色化

保留前景轮廓后，将 Logo 主体统一替换为目标色：

- 默认红色：`#ef0000`

#### 第四步：裁切与导出

自动按非透明区域裁切边界，输出透明 PNG，供前端直接预览和用于画布渲染。

## 技术实现

### Python 图像处理脚本

采用 `Pillow` 实现基础图像处理：

- 读取 PNG/JPG
- 背景估算
- Alpha 掩码生成
- 单色输出
- 透明 PNG 导出

### Node 服务调用方式

后端通过 `child_process.execFile` 调用 Python 脚本：

1. 将前端传来的 Data URL 临时写入上传目录
2. 调用 Python 脚本生成处理结果
3. 读取输出 PNG
4. 再转回 Data URL 返回前端

## 前端交互设计

### 上传后即时处理

用户点击“添加logo”并选择图片后：

1. 在消息气泡中插入一条处理状态消息
2. 展示处理进度
3. 处理完成后在同一气泡中更新结果
4. 将处理后的图片加入上传素材列表

### 消息气泡展示文案

建议分阶段展示：

```text
正在处理 Logo：爱家空间.png
1/3 已接收图片
2/3 正在去背景
3/3 正在转单色印刷稿
```

成功后更新为：

```text
Logo 处理完成：爱家空间.png
已完成去背景、单色化、扁平化，可直接用于当前效果图。
```

并在该消息气泡中附带处理后的预览图。

## 当前范围

本次先落地基础可用版，处理目标为：

- 浅色背景 Logo
- 单色印刷预览
- 首页效果图联动

## 后续可继续增强

### 可选增强 1：颜色可选

让管理员或用户选择：

- 红色
- 蓝色
- 黑色
- 白色

### 可选增强 2：高精度抠图

当浅色背景识别不理想时，可再接入：

- rembg
- 更强的分割模型
- 或 AI 兜底增强

### 可选增强 3：SVG 矢量化

后续如需更贴近印刷生产，可以增加：

- 轮廓提取
- SVG 导出
- 更规范的 logo 上版文件

## 结论

方案 A 适合当前项目阶段：

- 开发成本低
- 输出稳定
- 能直接改善首页保护垫预览图中的 Logo 效果
- 不依赖 `gpt-image-2`

后续如遇到复杂背景、低清晰度、严重高光等特殊素材，再考虑增加 AI 增强链路作为兜底。
